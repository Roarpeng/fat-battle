## 一、项目现有可玩性机制全景扫描

通过 GraphFlow 扫描 `useGameStore.ts`（28KB/35 符号）、`BattleEffects.tsx`（17KB/12 符号）、`GameFeedback.tsx`（8KB/14 符号）、`monsters.ts` 等核心文件，项目已构建了完整的游戏化框架：

### 1.1 核心游戏循环

```
记录饮食 → 摄入热量映射为怪物回血
运动打卡 → 消耗热量映射为对怪物伤害  
击败怪物 → 获得金币 + XP + 等级提升
每日结算 → 连胜天数 + 成就解锁 + 任务刷新
```

## 三、竞品核心设计模式提炼

### 3.1 最有效的 5 大留存机制（按效果排序）

**1. Streak 连续天数 + 容错机制**（Duolingo 标杆）
- Duolingo 最强大的留存工具——"已连续学习 365 天"的损失厌恶
- Streak Freeze 用宝石购买，允许一天不学习不中断
- **几乎所有成功产品都采用此模式**
- 减肥大作战：**已有 Streak，但缺少容错机制**——用户一旦断一天，连续记录归零，流失风险极高

**2. 叙事/情感驱动 > 数值驱动**
- Zombies, Run! 的音频故事比纯 XP 更有吸引力
- Wokamon 的宠物情感羁绊比步数排名更持久
- Ring Fit 的 RPG 冒险比"做 50 个深蹲"更有动力
- 减肥大作战：**有嘲讽/鼓励文字，但没有主线叙事**

**3. 社交竞争的低门槛化**
- Duolingo League：自动分组，无需主动社交
- Keep 社区：发动态就有反馈
- Habitica Party：团队责任互相约束
- 减肥大作战：**社交完全空白**

**4. 收集/图鉴驱动**
- Pokemon Sleep 的宝可梦图鉴
- Ring Fit 的装备收集
- Wokamon 的宠物进化
- 减肥大作战：**12 种怪物有定义，但没有"怪物图鉴"展示/收集进度**

**5. 渐进式难度 + 个性化**
- Ring Fit 从轻到重的自然过渡
- Keep 的 AI 教练个性化推荐
- 减肥大作战：**3 档难度 + BMI 自适应已有，但缺运动能力评估前置**

### 3.2 变现模式启示

| 模式 | 代表 | 适配度 | 说明 |
|------|------|--------|------|
| Freemium + 订阅 | Zombies, Run! / Duolingo | ⭐⭐⭐⭐⭐ | 解锁高级怪物皮肤、AI 教练、专属 BOSS |
| Freemium + IAP | Wokamon | ⭐⭐⭐⭐ | 特殊道具/技能/怪物皮肤 |
| 买断制 | Ring Fit | ⭐⭐ | 不适合 App 分发模式 |
| 混合（会员+内容） | Keep | ⭐⭐⭐ | 需要持续产出内容 |
| X-to-Earn | StepN | ❌ | 已验证不可持续，反面教材 |

## 四、可玩性提升建议（按 ROI 排序）

### 4.1 🔴 高 ROI — 快速见效

#### A. 添加 Streak 容错机制（1-2 天开发）

**对标**：Duolingo Streak Freeze

**现状问题**：用户断一天就全部归零，流失率高。竞品数据表明容错机制可提升 7 日留存 15-20%。

**建议方案**：
- 每周允许 1 次"补签"（免费）
- 可用金币购买额外补签次数（软付费点）
- 连续 7 天以上给予"护盾"自动保护 1 天

```ts
// store 扩展
interface StreakState {
  streak: number
  maxStreak: number          // 历史最长
  streakFreeze: number       // 可用冷冻次数
  lastActiveDate: string
  streakShield: boolean       // 7天以上自动护盾
}
```

#### B. 怪物图鉴系统（2-3 天开发）

**对标**：Pokemon Sleep 收集图鉴

**现状**：12 种怪物已定义（含赛季限定），但没有收集展示界面。

**建议**：
- 新增 `/codex` 页面，展示所有已遇到/击败的怪物
- 未遇到的显示为剪影 + 提示（"完成 Level X 解锁"）
- 怪物卡面展示：名称/属性/弱点/击败次数/首次击败日期
- 赛季限定怪物的稀有感：只在特定月份出现

#### C. 简单社交：好友排行榜（3-5 天开发）

**对标**：Duolingo League 的自动分组

**建议**：
- 第一阶段：仅好友间"本周运动量排行"，无需真实社交账号
- 第二阶段：每周自动分组（10 人一组），前 3 名晋级
- 数据维度：本周总伤害 / 本周运动时长 / 本周连胜天数
- **关键**：即使没有真实好友也可以和"随机对手"竞争

### 4.2 🟡 中 ROI — 差异化提升

#### D. 怪物叙事系统（1-2 周开发）

**对标**：Zombies, Run! 的音频叙事

**建议**：
- 为每种怪物添加一段简单的背景故事（击败后解锁）
- BOSS 战前触发一段"挑衅独白"（已有嘲讽框架，扩展为完整叙事）
- 击败 BOSS 后解锁下一段剧情
- 赛季限定怪物绑定季节故事（如"春季樱花精灵为何入侵你的身体"）

```
// 叙事触发点
const STORY_TRIGGERS = {
  first_encounter: '首次遇到此怪物时',
  phase_change: '怪物进入新阶段时',
  enrage: '怪物狂暴时',
  victory: '击败怪物时',
  first_kill: '首次击败此种类时',
}
```

#### E. 虚拟宠物/伙伴系统（2-3 周开发）

**对标**：Wokamon 宠物养成

**建议**：
- 解锁一只"战斗伙伴"（如小猫咪），跟随玩家成长
- 喂食 = 记录饮食，运动 = 伙伴获得 XP
- 伙伴等级解锁新外观/技能
- 伙伴在 BattlePage 上以小头像显示，在怪物攻击时有保护动画

**核心价值**：建立情感连接 > 纯数值激励。Wokamon 证明宠物养成是低门槛游戏化的最佳载体。

#### F. 动态难度 AI 教练（1-2 周开发）

**对标**：Keep 9.0 AI 教练"卡卡"

**建议**：
- 根据用户历史运动数据，自动调整：
  - 怪物血量（近期运动少 → 降 HP）
  - 任务目标（近期完成率低 → 降目标）
  - 每日任务类型（近期不做某类运动 → 推送引导任务）
- 在 BattlePage 上用伙伴对话形式给出建议（"主人~ 今天还没做核心运动哦"）

### 4.3 🟢 长期规划 — 战略差异化

#### G. 多人协作 Boss 战（1-2 月）

**对标**：Habitica 组队 Boss 战

**建议**：
- 好友可以组队打 Boss
- 每个人的运动/饮食贡献汇总为团队伤害
- 有人偷吃（超标饮食）会对全队造成反弹伤害
- 社交压力 + 团队荣誉 = 强留存

#### H. 怪物繁殖/融合系统（长期）

**对标**：Monster Legends 繁殖系统

**建议**：
- 击败精英/BOSS 有概率掉落"怪物碎片"
- 收集足够碎片可以合成/进化新怪物
- 不同怪物的碎片组合产出不同新怪物
- 为长期玩家提供收集驱动力

#### I. 赛季主题活动（持续运营）

**对标**：Pokemon Sleep 限定活动 + Zombies, Run! 赛季

**建议**：
- 每季度一个主题活动：新怪物线、限定成就、专属道具
- 赛季排行榜，积分兑换限定外观
- "减肥奥运会"——夏季主题赛季活动

## 五、竞品反面教训（必须避免）

### 5.1 StepN 的教训：经济激励不能替代内在动机
- 纯"赚钱"驱动在收益下降时用户瞬间流失
- **启示**：金币/道具可以作为点缀，但不能是核心驱动力

### 5.2 Fitocracy 的教训：不迭代就会被淘汰
- XP+徽章系统初期新颖，但长期没有深度
- **启示**：必须持续添加新内容（新怪物/新机制/新活动）

### 5.3 Wokamon 的教训：游戏深度不足
- 宠物养成有趣，但没有"第二核心循环"导致新鲜感消退
- **启示**：战斗系统（已精良）+ 养成系统（待建设）= 双核心循环

## 六、核心竞争优势与差异化定位

### 6.1 减肥大作战的独特优势

| 优势 | 对比说明 |
|------|----------|
| **饮食+运动统一对抗叙事** | 市场唯一将饮食记录和运动打卡统一到"打怪"的产品 |
| **BOSS 设计系统精良度** | 克制系统+多阶段+狂暴+赛季限定，复杂度接近主机游戏 |
| **体感动作识别** | MediaPipe 姿态识别实现真实运动→游戏动作映射 |
| **可爱宠物风格** | 差异化于 Keep 的"硬核健身"和 Zombies, Run! 的"末日叙事" |
| **PWA 离线支持** | 零安装门槛，扫码即用 |

### 6.2 建议差异化定位

> **"最可爱的怪物战斗健身伙伴"**
> 
> 不是硬核健身工具（Keep），不是恐怖生存（Zombies, Run!），不是可爱散步（Wokamon），而是 **用打怪的方式让你笑着减脂**。
