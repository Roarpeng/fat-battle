# 减肥大作战 - UI 重新设计计划

## 1. Summary

本次计划对"减肥大作战"Web App 进行重大 UI 改版，将脂肪怪推到视觉核心位置，简化导航结构，优化核心交互路径。目标：让用户打开 App 第一眼就看到敌人（脂肪怪），一键进入战斗（饮食/锻炼），操作后立刻看到敌人损耗。

## 2. Current State Analysis

### 2.1 导航结构（5 Tab）
- 当前底部导航：战斗 / 饮食 / 锻炼 / 统计 / 更多
- 问题：tab 过多，核心路径被稀释，用户需要思考"要去哪个tab"

### 2.2 首页（BattlePage）
- 当前：怪物卡片 + 玩家HP + 今日数据 + 快速操作（饮食/锻炼/统计入口）+ 底部提示
- 问题：信息过载，怪物不够突出，快速操作抢了主视觉

### 2.3 饮食页（FoodPage）
- 当前：三餐切换 + 搜索 + 食物列表 + 自定义食物 + 底部怪物HP条
- 问题：没有怪物陪伴感，AI识别入口不够突出

### 2.4 锻炼页（ExercisePage）
- 当前：运动类型列表 + 伤害预览 + 攻击按钮
- 问题：没有怪物陪伴感，摄像头检测入口需要跳转到独立页面

### 2.5 设置页（SettingsPage）
- 当前：角色信息 + 难度 + 其他设置 + 关于 + 游戏中心
- 问题：缺少食物/锻炼的自定义管理

### 2.6 数据文件
- `foods.ts`：预定义食物数组，无自定义食物持久化
- `exercises.ts`：预定义运动数组，无自定义运动持久化

## 3. Proposed Changes

### 3.1 导航重构：2 Tab 精简设计

**建议依据**：根据 Fitts 定律和 Nielsen 的可用性启发式原则，核心操作路径应最短。减肥 App 的核心循环只有"看怪物 -> 饮食/锻炼 -> 看怪物损耗"。统计是低频查看但重要反馈，设置是极低频操作。

**修改文件**：`components/MainLayout.tsx`
- 底部导航精简为 2 个 Tab：**首页**（脂肪怪图标）+ **统计**（图表图标）
- 首页右上角添加齿轮图标（设置入口）
- 移除"饮食""锻炼"独立 Tab（它们将通过首页按钮进入）

### 3.2 首页（BattlePage） redesign

**布局**：上 2/3 怪物区域，下 1/3 操作区域

**修改文件**：`pages/BattlePage.tsx`
- **怪物区域（上 2/3）**：
  - 顶部显示怪物血条（HpBar 组件）
  - 中间放大显示 MonsterCard（更大 emoji + 名称 + 等级）
  - 保留嘲讽台词气泡系统
  - 保留伤害/治疗飘字动画
- **操作区域（下 1/3）**：
  - 仅保留两个大按钮：**饮食**（左侧，红色系）+ **锻炼**（右侧，蓝色系）
  - 按钮带图标（Utensils + Dumbbell），大尺寸（占满底部区域）
  - 按钮文案下方显示今日简要数据（如"已记录 2 餐 / 已运动 1 次"）
- 移除：玩家 HP 条、今日数据详情卡片、快速操作行、底部提示（这些信息移到统计页）
- 添加右上角齿轮图标跳转到 SettingsPage

### 3.3 饮食页（FoodPage） redesign

**布局**：怪物 2/5 + 拍照/扫码 1/5 + 经常吃的食物 2/5

**修改文件**：`pages/FoodPage.tsx`

- **拍照/扫码区域（中 1/5）**：

### 3.4 锻炼页（ExercisePage） redesign

## 4. Assumptions & Decisions

### 4.1 导航设计决策
- **2 Tab 而非 3 Tab**：首页和统计是唯二需要常驻导航的页面。设置通过首页齿轮图标进入，符合低频操作不占用主空间的 UX 原则。
- **饮食/锻炼不再作为独立 Tab**：它们通过首页大按钮进入，形成"首页 -> 子页面 -> 首页"的闭环，比 Tab 切换更符合游戏化流程。

### 4.2 怪物缩放设计
- **MiniMonsterCard 组件**：在 FoodPage/ExercisePage 上保持怪物的"存在感"，让用户始终记得自己在和怪物战斗。血条变化提供即时反馈。

### 4.3 经常吃的食物排序逻辑
- 基于 `dietRecords` 历史数据，按出现频率统计 Top 8。如果记录不足 8 条，用默认食物（含杭州特色）填充。

### 4.4 自定义数据持久化
- 自定义食物和运动存在 game store 中，通过 Zustand persist 自动保存到 localStorage。不需要额外后端。

### 4.5 杭州特色食物选择
- 选择热量数据明确、在当地有代表性的食物：片儿川、东坡肉、龙井虾仁、西湖醋鱼、葱包桧、杭州小笼包、定胜糕、西湖藕粉、猫耳朵、西湖莼菜汤。

## 5. Verification Steps

1. **导航验证**：底部只有 2 个 tab（首页/统计），设置通过齿轮图标进入
2. **首页验证**：上 2/3 是怪物（血条在顶），下 1/3 只有饮食/锻炼两个大按钮
3. **饮食页验证**：顶部怪物 2/5，中间拍照入口 1/5，下方经常吃的食物 2/5
4. **锻炼页验证**：顶部怪物 2/5，中间摄像头/手动按钮 1/5，下方自定义运动 2/5
5. **设置页验证**：包含食物自定义、运动自定义、蓝牙链接
6. **杭州食物验证**：FoodPage 默认显示杭州特色食物
7. **自定义持久化验证**：添加自定义食物/运动后刷新页面，数据不丢失
8. **TypeScript 编译验证**：`npx tsc --noEmit` 零错误
9. **浏览器验证**：所有页面布局正确，跳转流畅，无报错
